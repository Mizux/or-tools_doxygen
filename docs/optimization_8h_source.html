35 // Tries to minimize the given UNSAT core with a really simple heuristic.
36 // The idea is to remove literals that are consequences of others in the core.
37 // We already know that in the initial order, no literal is propagated by the
38 // one before it, so we just look for propagation in the reverse order.
39 //
40 // Important: The given SatSolver must be the one that just produced the given
41 // core.
42 void MinimizeCore(SatSolver* solver, std::vector<Literal>* core);
43 
44 // Like MinimizeCore() with a slower but strictly better heuristic. This
45 // algorithm should produce a minimal core with respect to propagation. We put
46 // each literal of the initial core "last" at least once, so if such literal can
47 // be inferred by propagation by any subset of the other literal, it will be
48 // removed.
49 //
50 // Note that this function doest NOT preserve the order of Literal in the core.
51 void MinimizeCoreWithPropagation(SatSolver* solver, std::vector<Literal>* core);

59 // All the Solve*() functions below reuse the SatSolver::Status with a slightly
60 // different meaning:
61 // - FEASIBLE: The problem has been solved to optimality.
62 // - INFEASIBLE: Same meaning, the decision version is already unsat.
63 // - LIMIT_REACHED: we may have some feasible solution (if solution is
64 // non-empty), but the optimality is not proven.

66 // Implements the "Fu & Malik" algorithm described in:
67 // Zhaohui Fu, Sharad Malik, "On solving the Partial MAX-SAT problem", 2006,
68 // International Conference on Theory and Applications of Satisfiability
69 // Testing. (SAT’06), LNCS 4121.
70 //
71 // This algorithm requires all the objective weights to be the same (CHECKed)
72 // and currently only works on minimization problems. The problem is assumed to
73 // be already loaded into the given solver.
74 //
75 // TODO(user): double-check the correctness if the objective coefficients are
76 // negative.
77 SatSolver::Status SolveWithFuMalik(LogBehavior log,
78  const LinearBooleanProblem& problem,
79  SatSolver* solver,
80  std::vector<bool>* solution);
81 
82 // The WPM1 algorithm is a generalization of the Fu & Malik algorithm to
83 // weighted problems. Note that if all objective weights are the same, this is
84 // almost the same as SolveWithFuMalik() but the encoding of the constraints is
85 // slightly different.
86 //
87 // Ansotegui, C., Bonet, M.L., Levy, J.: Solving (weighted) partial MaxSAT
88 // through satisﬁability testing. In: Proc. of the 12th Int. Conf. on Theory and
89 // Applications of Satisﬁability Testing (SAT’09). pp. 427-440 (2009)
90 SatSolver::Status SolveWithWPM1(LogBehavior log,
91  const LinearBooleanProblem& problem,
92  SatSolver* solver, std::vector<bool>* solution);

103 // Starts by solving the decision version of the given LinearBooleanProblem and
104 // then simply add a constraint to find a lower objective that the current best
105 // solution and repeat until the problem becomes unsat.
106 //
107 // The problem is assumed to be already loaded into the given solver. If
108 // solution is initially a feasible solution, the search will starts from there.
109 // solution will be updated with the best solution found so far.
110 SatSolver::Status SolveWithLinearScan(LogBehavior log,
111  const LinearBooleanProblem& problem,
112  SatSolver* solver,
113  std::vector<bool>* solution);
