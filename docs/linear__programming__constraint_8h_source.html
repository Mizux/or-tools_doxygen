52 // A SAT constraint that enforces a set of linear inequality constraints on
53 // integer variables using an LP solver.
54 //
55 // The propagator uses glop's revised simplex for feasibility and propagation.
56 // It uses the Reduced Cost Strengthening technique, a classic in mixed integer
57 // programming, for instance see the thesis of Tobias Achterberg,
58 // "Constraint Integer Programming", sections 7.7 and 8.8, algorithm 7.11.
59 // http://nbn-resolving.de/urn:nbn:de:0297-zib-11129
60 //
61 // Per-constraint bounds propagation is NOT done by this constraint,
62 // it should be done by redundant constraints, as reduced cost propagation
63 // may miss some filtering.
64 //
65 // Note that this constraint works with double floating-point numbers, so one
66 // could be worried that it may filter too much in case of precision issues.
67 // However, by default, we interpret the LP result by recomputing everything
68 // in integer arithmetic, so we are exact.
69 class LinearProgrammingDispatcher;

116  // Returns a LiteralIndex guided by the underlying LP constraints.

127  // Returns a LiteralIndex guided by the underlying LP constraints.
128  // This computes the mean of reduced costs over successive calls,
129  // and tries to fix the variable which has the highest reduced cost.
130  // Tie-breaking is done using the variable natural order.
131  // Only works for 0/1 variables.
132  //
133  // TODO(user): Try to get better pseudocosts than averaging every time
134  // the heuristic is called. MIP solvers initialize this with strong branching,
135  // then keep track of the pseudocosts when doing tree search. Also, this
136  // version only branches on var >= 1 and keeps track of reduced costs from var
137  // = 1 to var = 0. This works better than the conventional MIP where the
138  // chosen variable will be argmax_var min(pseudocost_var(0->1),
139  // pseudocost_var(1->0)), probably because we are doing DFS search where MIP
140  // does BFS. This might depend on the model, more trials are necessary. We
141  // could also do exponential smoothing instead of decaying every N calls, i.e.
142  // pseudo = a * pseudo + (1-a) reduced.
143  std::function<LiteralIndex()> HeuristicLPPseudoCostBinary(Model* model);
