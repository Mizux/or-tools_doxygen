14 // This file contains all the top-level logic responsible for driving the search
15 // of a satisfiability integer problem. What decision we take next, which new
16 // Literal associated to an IntegerLiteral we create and when we restart.
17 //
18 // For an optimization problem, our algorithm solves a sequence of decision
19 // problem using this file as an entry point. Note that some heuristics here
20 // still use the objective if there is one in order to orient the search towards
21 // good feasible solution though.

35 // Model struct that contains the search heuristics used to find a feasible
36 // solution to an integer problem.
37 //
38 // This is reset by ConfigureSearchHeuristics() and used by
39 // SolveIntegerProblem(), see below.
40 struct SearchHeuristics {
41  // Decision and restart heuristics. The two vectors must be of the same size
42  // and restart_policies[i] will always be used in conjunction with
43  // decision_policies[i].
44  std::vector<std::function<LiteralIndex()>> decision_policies;
45  std::vector<std::function<bool()>> restart_policies;
46 
47  // Index in the vector above that indicate the current configuration.
48  int policy_index;
49 };

58 // For an optimization problem, this contains the internal integer objective
59 // to minimize and information on how to display it correctly in the logs.
60 struct ObjectiveDefinition {

63  IntegerVariable objective_var = kNoIntegerVariable;

65  double ScaleIntegerObjective(IntegerValue value) const {

76 // Tries to find a feasible solution to the current model.
77 //
78 // This function continues from the current state of the solver and loop until
79 // all variables are instantiated (i.e. the next decision is kNoLiteralIndex) or
80 // a search limit is reached. It uses the heuristic from the SearchHeuristics
81 // class in the model to decide when to restart and what next decision to take.
82 //
83 // Each time a restart happen, this increment the policy index modulo the number
84 // of heuristics to act as a portfolio search.
85 SatSolver::Status SolveIntegerProblem(Model* model);

159 // Changes the value of the given decision by 'var_selection_heuristic'. We try
160 // to see if the decision is "associated" with an IntegerVariable, and if it is
161 // the case, we choose the new value by the first 'value_selection_heuristics'
162 // that is applicable (return value != kNoLiteralIndex). If none of the
163 // heuristics are applicable then the given decision by
164 // 'var_selection_heuristic' is returned.

170 // Changes the value of the given decision by 'var_selection_heuristic'. The new
171 // value is chosen in the following way:
172 // 1) If the LP part is large and the LP solution is exploitable, change the
173 // value to LP solution value.
174 // 2) Else if there is at least one solution found, change the value to the best
175 // solution value.
176 // If none of the above heuristics are applicable then it uses the default value
177 // given in the decision.
