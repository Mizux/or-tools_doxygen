28 // Overload Checker
29 //
30 // This propagator implements the overload checker filtering rule presented in
31 // Vilim Petr, "Max Energy Filtering Algorithm for Discrete Cumulative
32 // Constraint", CPAIOR 2009, http://vilim.eu/petr/cpaior2009.pdf.
33 //
34 // This propagator only increases the minimum of the capacity variable or fails
35 // if the minimum capacity cannot be increased. It has a time complexity of
36 // O(n log n).
37 //
38 // The propagator relies on a Theta-tree to maintain the energy and envelope of
39 // several set of tasks.
40 //
41 // The energy of a task can be seen as its surface and is the product of its
42 // minimum demand and minimum duration. The energy of a set of tasks is the sum
43 // of the energy of its tasks.
44 //
45 // The envelope of a task is the sum of the task energy and the total amount of
46 // energy available before the minimum starting time of the task, i.e., the
47 // product of its minimum start time by the maximum capacity of the resource.
48 //
49 // An overload, meaning that there is no solution, occurs when a set of tasks
50 // requires more energy than what is available between its starting and ending
51 // times.
52 //
53 // Be aware that overload checker is not enough to ensure that the cumulative
54 // constraint holds. This propagator should thus always be used with a
55 // timetabling propagator at least.
