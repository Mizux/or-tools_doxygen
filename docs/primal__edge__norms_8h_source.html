28 // This class maintains the primal edge squared norms (and other variants) to be
29 // used in the primal pricing step. Instead of computing the needed values from
30 // scractch at each iteration, it is more efficient to update them incrementally
31 // for each basis pivot applied to the simplex basis matrix B.
32 //
33 // Terminology:
34 // - To each non-basic column 'a' of a matrix A, we can associate an "edge" in
35 // the kernel of A equal to 1.0 on the index of 'a' and '-B^{-1}.a' on the
36 // basic variables.
37 // - 'B^{-1}.a' is called the "right inverse" of 'a'.
38 // - The entering edge is the edge we are following during a simplex step,
39 // and we call "direction" the reverse of this edge restricted to the
40 // basic variables, i.e. the right inverse of the entering column.
41 //
42 // Papers:
43 // - D. Goldfarb, J.K. Reid, "A practicable steepest-edge simplex algorithm"
44 // Mathematical Programming 12 (1977) 361-371, North-Holland.
45 // http://www.springerlink.com/content/g8335137n3j16934/
46 // - J.J. Forrest, D. Goldfarb, "Steepest-edge simplex algorithms for linear
47 // programming", Mathematical Programming 57 (1992) 341-374, North-Holland.
48 // http://www.springerlink.com/content/q645w3t2q229m248/
49 // - Ping-Qi Pan "A fast simplex algorithm for linear programming".
50 // http://www.optimization-online.org/DB_FILE/2007/10/1805.pdf
51 // - Ping-Qi Pan, "Efficient nested pricing in the simplex algorithm",
52 // http://www.optimization-online.org/DB_FILE/2007/10/1810.pdf

73  // Returns the primal edge squared norms. This is only valid if the caller

76  // relevant columns are filled.
77  const DenseRow& GetEdgeSquaredNorms();

79  // Returns an approximation of the edges norms "devex".

82  // after a Clear().
83  const DenseRow& GetDevexWeights();
84 
85  // Returns the L2 norms of all the columns of A.
86  // Note that this is currently not cleared by Clear().
87  const DenseRow& GetMatrixColumnNorms();
88 
89  // Compares the current entering edge norm with its precise version (using the
90  // direction that wasn't avaible before) and triggers a full recomputation if
91  // the precision is not good enough (see recompute_edges_norm_threshold in
92  // GlopParameters). As a side effect, this replace the entering_col edge
93  // norm with its precise version.
94  void TestEnteringEdgeNormPrecision(ColIndex entering_col,
95  const ScatteredColumn& direction);
96 
97  // Updates any internal data BEFORE the given simplex pivot is applied to B.
98  // Note that no updates are needed in case of a bound flip.
99  // The arguments are in order:
100  // - The index of the entering non-basic column of A.
101  // - The index in B of the leaving basic variable.
102  // - The 'direction', i.e. the right inverse of the entering column.
103  // - The update row (see UpdateRow), which will only be computed if needed.
104  void UpdateBeforeBasisPivot(ColIndex entering_col, ColIndex leaving_col,
105  RowIndex leaving_row,
106  const ScatteredColumn& direction,
107  UpdateRow* update_row);
