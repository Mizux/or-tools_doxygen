61 // Given an automaton defined by a set of 3-tuples:
62 // (state, transition_with_value_as_label, next_state)
63 // this accepts the sequences of vars.size() variables that are recognized by
64 // this automaton. That is:
65 // - We start from the initial state.
66 // - For each variable, we move along the transition labeled by this variable
67 // value. Moreover, the variable must take a value that correspond to a
68 // feasible transition.
69 // - We only accept sequences that ends in one of the final states.
70 //
71 // We CHECK that there is only one possible transition for a state/value pair.
72 // See the test for some examples.
73 std::function<void(Model*)> TransitionConstraint(
74  const std::vector<IntegerVariable>& vars,
75  const std::vector<std::vector<int64>>& automaton, int64 initial_state,
76  const std::vector<int64>& final_states);
