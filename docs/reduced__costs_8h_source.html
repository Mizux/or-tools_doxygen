31 // Maintains the reduced costs of the non-basic variables and some related
32 // quantities.
33 //
34 // Terminology:
35 // - To each non-basic column 'a' of A, we can associate an "edge" in the
36 // kernel of A equal to 1.0 on the index of 'a' and '-B^{-1}.a' on the basic
37 // variables.
38 // - 'B^{-1}.a' is called the "right inverse" of 'a'.
39 // - The reduced cost of a column is equal to the scalar product of this
40 // column's edge with the cost vector (objective_), and corresponds to the
41 // variation in the objective function when we add this edge to the current
42 // solution.
43 // - The dual values are the "left inverse" of the basic objective by B.
44 // That is 'basic_objective_.B^{-1}'
45 // - The reduced cost of a column is also equal to the scalar product of this
46 // column with the vector of the dual values.
47 class ReducedCosts {

68  // Computes the current dual residual and infeasibility. Note that these
69  // functions are not really fast (many scalar products will be computed) and
70  // shouldn't be called at each iteration. They will return 0.0 if the reduced

76  // Updates any internal data BEFORE the given simplex pivot is applied to B.
77  // Note that no updates are needed in case of a bound flip.
78  // The arguments are in order:
79  // - The index of the entering non-basic column of A.
80  // - The index in B of the leaving basic variable.
81  // - The 'direction', i.e. the right inverse of the entering column.
82  void UpdateBeforeBasisPivot(ColIndex entering_col, RowIndex leaving_row,
83  const ScatteredColumn& direction,
84  UpdateRow* update_row);

86  // Once a pivot has been done, this need to be called on the column that just
87  // left the basis before the next ChooseEnteringColumn(). On a bound flip,
88  // this must also be called with the variable that flipped.
89  //
90  // In both cases, the variable should be dual-feasible by construction. This
91  // sets is_dual_infeasible_[col] to false and checks in debug mode that the
92  // variable is indeed not an entering candidate.
93  void SetAndDebugCheckThatColumnIsDualFeasible(ColIndex col);

95  // Sets the cost of the given non-basic variable to zero and updates its
96  // reduced cost. Note that changing the cost of a non-basic variable only
97  // impacts its reduced cost and not the one of any other variables.
98  // The current_cost pointer must be equal to the address of objective[col]
99  // where objective is the DenseRow passed at construction.

103  void SetParameters(const GlopParameters& parameters);

105  // Returns true if the current reduced costs are computed with maximum
106  // precision.
107  bool AreReducedCostsPrecise() { return are_reduced_costs_precise_; }

109  // Returns true if the current reduced costs where just recomputed or will be

113  }

118  void MakeReducedCostsPrecise();

120  // Randomly perturb the costs. Both Koberstein and Huangfu recommend doing
121  // that before the dual simplex starts in their Phd thesis.
122  //
123  // The perturbation follows what is explained in Huangfu Q (2013) "High
124  // performance simplex solver", Ph.D, dissertation, University of Edinburgh,
125  // section 3.2.3, page 58.
126  void PerturbCosts();
127 
128  // Shifts the cost of the given non-basic column such that its current reduced
129  // cost becomes 0.0. Actually, this shifts the cost a bit more, so that the
130  // reduced cost becomes slightly of the other sign.
131  //
132  // This is explained in Koberstein's thesis (section 6.2.2.3) and helps on
133  // degenerate problems. As of july 2013, this allowed to pass dano3mip and
134  // dbic1 without cycling forever. Note that contrary to what is explained in
135  // the thesis, we do not shift any other variable costs. If any becomes
136  // infeasible, it will be selected and shifted in subsequent iterations.
137  void ShiftCost(ColIndex col);
138 
139  // Removes any cost shift and cost perturbation. This also lazily forces a
140  // recomputation of all the derived quantities. This effectively resets the
141  // class to its initial state.
142  void ClearAndRemoveCostShifts();
143 
144  // Invalidates all internal structure that depends on the objective function.
145  void ResetForNewObjective();

147  // Sets whether or not the bitset of the dual infeasible positions is updated.
148  void MaintainDualInfeasiblePositions(bool maintain);
149 
150  // Invalidates the data that depends on the order of the column in basis_.
151  void UpdateDataOnBasisPermutation();
152 
153  // Returns the current reduced costs. If AreReducedCostsPrecise() is true,
154  // then for basic columns, this gives the error between 'c_B' and 'y.B' and
155  // for non-basic columns, this is the classic reduced cost. If it is false,
156  // then this is defined only for the columns in
157  // variables_info_.GetIsRelevantBitRow().
158  const DenseRow& GetReducedCosts();
